Q1: How does JavaScript work?
  1. Javascript run as interpreted programming language. Means, Javascript translates one statement at a time.
  2. JavaScript is a high level dynamic typed programming language which means in javascript we not need to declear the type of variable.
  3. In chrome javaScript code run using v8 engine.
  4. V8 engine compiles JavaScript code into machine code using Just-In-Time compiler.
  5. V8 engine takes javaScript source code and convert into Abstract Syntax Tree.
  6. Then bytecode is interpreted by a Ignition interpreter.
  7. Then TurboFan optimize and Compiles the code into an optimized Machine language.
Q2: What does JavaScript's event loop do?
  1. JavaScript is synchronous and single threaded programming Language. Means JavaScript excute code through line by line synchronously.
  2. By using Event Loop JavaScript can execute code asynchronously.
  3. The Event Loop is a constantly running process that handle Call Stack and Callback Queue.
  4. In event loop Call Stack is synchronous and Callback Queue is an asynchronous/non-blocking.
  5. In event loop all synchronous event executed by Call Stack.
  6. When an asynchronous event is found JavaScript push the event in Callback Queue.
  7. After executing all synchronous event javaScript take each asynchronous event from the queue of Callback Queue.
  8. Then Call Stack execute each event in the queue.
  9. Asynchronous functions are: setTimeout, setInterval, fetch, async await.
Q3: Local Storage vs Session Storage?
  1. In Local Storage data is not clean from the browser even when the tab or the whole browser is closed.
  2. There is no data expiration date for Local Storage stored data.
  3. So, Storing sensitive data in Local Storage can be a threat. Because if the hackers want they can access those stored data.
  4. Session storage is a good choice for storing data in the browser. Because Session Storage only contains data for a specific session.
  5. In Session Storage stored data is cleared when browser tab is closed or session is ends.
  6. Since data is cleared in Session Storage after closing the tab, there is no need to compromise with browser performance.
